+==============================================================================================+
|                              核心业务流程 (Business Processes)                                |
+==============================================================================================+

  【流程1: VPC 创建业务流程】

  +----------+     +----------+     +----------+     +----------+     +----------+
  |  租户    |     |  接入    |     |  编排    |     |  执行    |     |  交付    |
  |  申请    | --> |  验证    | --> |  规划    | --> |  配置    | --> |  确认    |
  +----------+     +----------+     +----------+     +----------+     +----------+
       |               |               |               |               |
       v               v               v               v               v
  +---------+     +---------+     +---------+     +---------+     +---------+
  | 提交VPC |     | 参数校验|     | 确定区域|     | 并行下发|     | 状态同步|
  | 创建请求|     | 配额检查|     | 分配资源|     | 各AZ执行|     | 结果通知|
  +---------+     +---------+     +---------+     +---------+     +---------+

  业务规则:
  - Region级服务: VPC在指定Region的所有AZ并行创建
  - 原子性保证: 任一AZ失败则全部回滚
  - 幂等性设计: 支持重试不产生副作用


  【流程2: 子网 创建业务流程】

  +----------+     +----------+     +----------+     +----------+
  |  租户    |     |  关联    |     |  地址    |     |  配置    |
  |  申请    | --> |  校验    | --> |  分配    | --> |  生效    |
  +----------+     +----------+     +----------+     +----------+
       |               |               |               |
       v               v               v               v
  +---------+     +---------+     +---------+     +---------+
  | 指定VPC |     | VPC存在 |     | CIDR规划|     | 单AZ执行|
  | 指定AZ  |     | AZ可用  |     | IP池分配|     | 路由配置|
  +---------+     +---------+     +---------+     +---------+

  业务规则:
  - AZ级服务: 子网在指定的单个AZ创建
  - 依赖检查: 必须先存在对应的VPC
  - 地址规划: CIDR不可与同VPC其他子网重叠


  【流程3: 防火墙策略 创建业务流程】

  +----------+     +----------+     +----------+     +----------+     +----------+
  |  安全    |     |  信息    |     |  范围    |     |  策略    |     |  生效    |
  |  申请    | --> |  解析    | --> |  确定    | --> |  下发    | --> |  验证    |
  +----------+     +----------+     +----------+     +----------+     +----------+
       |               |               |               |               |
       v               v               v               v               v
  +---------+     +---------+     +---------+     +---------+     +---------+
  | 源/目IP |     | IP->Zone|     | 同AZ/   |     | 并行配置|     | 策略生效|
  | 端口协议|     | 映射查询|     | 跨AZ判断|     | 防火墙  |     | 状态确认|
  +---------+     +---------+     +---------+     +---------+     +---------+

  业务规则:
  - 智能路由: 根据IP自动识别所属Zone和AZ
  - 跨域策略: 跨AZ通信需在两端AZ同时配置
  - 最小权限: 默认拒绝，显式允许
