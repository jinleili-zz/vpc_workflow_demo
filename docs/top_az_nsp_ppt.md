# 一页PPT：TOP-NSP 设计逻辑

**标题建议**：Top NSP（全局编排层）

**核心定位**
- 全局编排与管控入口：统一接入 VPC/区域级请求，负责跨 AZ 并行编排。
- AZ 注册与健康管理：维护 AZ 注册表与心跳，保证可用性与可观测性。

**主要功能**
- **区域级 VPC 编排**：并行向各 AZ 发起创建任务，统一聚合结果。
- **故障处理与回滚**：任一 AZ 失败时触发已成功 AZ 的自动回滚。
- **服务发现**：维护 AZ 列表、健康状态与服务地址，供请求路由。

**实现逻辑（可直接放流程图）**
1. 接收 Region 级请求（如创建 VPC）。
2. 读取 AZ 注册表，筛选可用 AZ。
3. 并行分发到各 AZ NSP，触发 AZ 内部工作流。
4. 聚合结果，若存在失败则触发回滚并返回统一状态。

**关键组件/技术要点**
- Orchestrator（并行编排与回滚）
- Registry（AZ 注册/心跳）
- Redis（状态存储 DB0 + 任务队列 DB1）

# 一页PPT：AZ-NSP 设计逻辑

**标题建议**：AZ NSP（区域可用区执行层）

**核心定位**
- AZ 内执行单元：承接 Top NSP 的任务下发，执行单 AZ 的工作流。
- 负责链式任务编排与执行：驱动交换机/防火墙等具体任务。

**主要功能**
- **AZ 级工作流执行**：按顺序执行 VPC/子网工作流（链式任务）。
- **任务队列隔离**：按 AZ 使用独立队列，避免跨 AZ 竞争。
- **状态记录与查询**：写入 Redis 供状态查询与追踪。

**实现逻辑（可直接放流程图）**
1. 启动时向 Top NSP 注册，并定时心跳上报。
2. 接收 Top NSP 的 AZ 级请求（VPC/子网）。
3. 创建链式任务（VRF → VLAN 子接口 → 防火墙区），逐步入队。
4. Worker 消费 AZ 专用队列，更新工作流状态到 Redis。

**关键组件/技术要点**
- AZ API（接入与注册）
- Workers（switch/firewall 等任务处理）
- Asynq 队列（AZ 专用队列）
- Redis（状态与任务）
