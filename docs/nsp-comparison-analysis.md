## 一、架构对比

|                |                        |                        |                        |
+----------------+------------------------+------------------------+------------------------+
|                |                        |                        |                        |

+----------------+------------------------+------------------------+------------------------+
|                |                        |                        |                        |
|   数据面       |  物理设备配置下发       |  分布式虚拟交换机       |  虚拟交换机             |
|                |  Switch/Firewall       |  软件网关集群           |  软件网关               |
|                |                        |                        |                        |
+----------------+------------------------+------------------------+------------------------+

**架构特点总结：**

| 系统 | 特点 |
|------|------|
| **NSP** | 轻量级两级架构，面向物理设备编排，适合私有云/混合云场景 |
| **阿里云** | 重度SDN架构，全软件定义，面向超大规模公有云 |
| **腾讯云** | SDN架构，虚拟化网络，面向公有云多租户场景 |

## 六、总结

| 维度 | NSP | 阿里云/腾讯云 |
|------|-----|---------------|
| **定位** | 私有云/企业级网络服务平台 | 公有云网络基础设施 |
| **架构** | 轻量两级微服务 | 重度SDN架构 |
| **数据面** | 物理设备配置下发 | 纯软件虚拟网络 |
| **部署** | 分钟级、最小依赖 | 需要完整云平台支撑 |
| **适用** | 已有物理网络设备的企业 | 纯虚拟化/容器化场景 |

**一句话总结：**
> NSP 是面向企业私有云场景的轻量级网络服务平台，通过开源技术栈实现物理网络设备的自动化编排，填补了公有云SDN方案无法对接物理设备的空白。
