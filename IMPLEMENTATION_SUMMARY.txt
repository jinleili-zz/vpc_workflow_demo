## 已实现功能

### 1. Top NSP (Global 层)
✅ Region/AZ 动态注册
✅ AZ 心跳管理和健康检查
✅ Region 级服务编排（VPC - 并行发送到所有 AZ）
✅ AZ 级服务路由（子网 - 发送到指定 AZ）
✅ RESTful API 接口

### 2. AZ NSP (Region/AZ 层)
✅ 自动注册到 Top NSP
✅ 定时心跳（60秒）
✅ VPC 工作流执行（VRF → VLAN → Firewall）
✅ 子网工作流执行（创建子网 → 配置路由）
✅ 任务状态查询（基于 VPC 名字）
✅ Worker 进程管理

### 3. 服务级别
✅ Region 级服务：VPC（在 Region 的所有 AZ 创建）
✅ AZ 级服务：子网（在指定 AZ 创建）

### 4. 容器化支持
✅ Dockerfile（Top NSP 和 AZ NSP）
✅ Docker Compose 配置
✅ 本地启动脚本
✅ 端到端测试脚本

## 技术特性

1. **动态注册**: AZ NSP 启动时自动注册，无需手动配置
2. **健康监控**: 心跳机制确保 AZ 状态实时更新
3. **并行执行**: Region 级服务并行发送到所有 AZ
4. **任务编排**: 基于 go-machinery 的 Chain 模式
5. **状态查询**: 基于用户友好标识符（VPC 名字）
6. **Redis 分离**: DB0（数据存储）、DB1（消息队列）
7. **容器化**: 支持 Docker Compose 一键部署

## 部署说明

### 本地部署
```bash
# 1. 编译
go build -o top-nsp ./cmd/top_nsp
go build -o az-nsp ./cmd/az_nsp

# 2. 启动
bash scripts/start-nsp-local.sh

# 3. 测试
bash scripts/test-e2e-local.sh

# 4. 停止
pkill -f nsp
```

### Docker 部署
```bash
# 1. 构建镜像
bash deployments/docker/build-images.sh

# 2. 启动
cd deployments/docker
docker-compose up -d

# 3. 测试
bash test-e2e.sh

# 4. 查看日志
docker-compose logs -f top-nsp
docker-compose logs -f az-nsp-cn-beijing-1a

# 5. 停止
docker-compose down
```

## API 使用示例

### 1. 创建 Region 级 VPC
```bash
curl -X POST http://localhost:8080/api/v1/vpc \
  -H "Content-Type: application/json" \
  -d '{
    "vpc_name": "my-vpc",
    "region": "cn-beijing",
    "vrf_name": "VRF-001",
    "vlan_id": 100,
    "firewall_zone": "trust-zone"
  }'
```

### 2. 创建 AZ 级子网
```bash
curl -X POST http://localhost:8080/api/v1/subnet \
  -H "Content-Type: application/json" \
  -d '{
    "subnet_name": "my-subnet",
    "vpc_name": "my-vpc",
    "region": "cn-beijing",
    "az": "cn-beijing-1a",
    "cidr": "10.0.1.0/24"
  }'
```

### 3. 查询 Region 的 AZ 列表
```bash
curl http://localhost:8080/api/v1/regions/cn-beijing/azs
```

### 4. 查询 VPC 状态
```bash
curl http://localhost:9081/api/v1/vpc/my-vpc/status
```
